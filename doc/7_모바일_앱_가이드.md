# 모바일 앱 (Android/iOS) 가이드

이 문서는 AdvanceKeep 웹 애플리케이션을 **Capacitor**를 사용하여 네이티브 모바일 앱으로 빌드하고 실행하는 방법을 설명합니다.

## 1. 개요
*   **Capacitor**는 웹 앱의 코드를 그대로 사용하여 네이티브 안드로이드 및 iOS 앱을 생성할 수 있게 해주는 크로스 플랫폼 런타임입니다.
*   웹 코드는 `dist` 폴더에 빌드되며, Capacitor가 이를 네이티브 프로젝트(`android`, `ios`) 내의 웹뷰(WebView)로 복사하여 앱을 구동합니다.

## 2. 사전 준비 사항
*   **Node.js**: 최신 LTS 버전
*   **Android Studio**: 안드로이드 앱 빌드 및 에뮬레이터 실행용 (Windows/Mac)
*   **Xcode**: iOS 앱 빌드 및 시뮬레이터 실행용 (**Mac 필수**)
    *   *주의: Windows 환경에서는 iOS 프로젝트 폴더(`.ios`) 생성까지만 가능하며, 실제 빌드는 불가능합니다.*

## 3. 앱 실행 방법

### 3.1 안드로이드 (Android)
1.  **웹 앱 빌드**:
    ```bash
    npm run build
    ```
2.  **Capacitor 동기화**:
    ```bash
    npx cap sync
    ```
3.  **Android Studio 열기**:
    ```bash
    npx cap open android
    ```
4.  **앱 실행**: Android Studio 상단의 **Run** 버튼(▶)을 클릭하여 에뮬레이터 또는 연결된 기기에서 앱을 실행합니다.

### 3.2 아이폰 (iOS)
1.  프로젝트 전체를 **Mac** 환경으로 복사합니다.
2.  **의존성 설치 및 빌드**:
    ```bash
    npm install
    npm run build
    npx cap sync
    ```
3.  **Xcode 열기**:
    ```bash
    npx cap open ios
    ```
4.  **앱 실행**: Xcode 상단에서 시뮬레이터 기종을 선택하고 **Run** 버튼(▶)을 클릭합니다.

## 4. 앱 업데이트 및 유지보수
웹 소스 코드(`src`)를 수정한 경우, 네이티브 앱에 변경 사항을 반영하려면 다음 과정을 반복해야 합니다:

1.  코드 수정 (React)
2.  `npm run build` (웹 자산 재생성)
3.  `npx cap sync` (네이티브 프로젝트로 자산 복사)
4.  Android Studio / Xcode에서 앱 재실행

## 5. 문제 해결
*   **동기화 오류**: `npx cap sync` 실행 시 오류가 발생하면 `dist` 폴더가 존재하는지(`npm run build` 실행 여부) 확인하세요.
*   **플러그인 오류**: 새로운 네이티브 기능(카메라 등)을 추가한 경우 `npx cap sync`를 다시 실행해야 네이티브 프로젝트에 반영됩니다.
