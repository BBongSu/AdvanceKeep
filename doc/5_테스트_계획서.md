# 테스트 계획서 (Test Plan)

## 1. 개요
본 문서는 AdvanceKeep 애플리케이션의 품질 보증을 위한 테스트 전략, 범위, 절차를 기술합니다.

## 2. 테스트 환경
-   **브라우저**: Chrome (최신 버전), Edge
-   **OS**: Windows 10/11
-   **네트워크**: 로컬 환경 (localhost)

## 3. 테스트 범위

### 3.1 단위 테스트 (Unit Test)
-   현재 별도의 자동화된 단위 테스트 코드는 존재하지 않음 (향후 Jest/Vitest 도입 고려).
-   유틸리티 함수(`src/utils`)의 주요 로직 검증.

### 3.2 통합 테스트 (Integration Test)
-   프론트엔드와 JSON Server 간의 데이터 연동 테스트.
-   API 요청 및 응답 처리 확인.

### 3.3 UI/UX 테스트 (Manual)
-   **화면 레이아웃**: 반응형 디자인이 정상적으로 작동하는지 확인.
-   **인터랙션**: 버튼 클릭, 모달 오픈/클로즈, 드래그 앤 드롭(구현 시) 등이 부드럽게 동작하는지 확인.

## 4. 테스트 시나리오 (Test Cases)

| TC ID | 테스트 항목 | 사전 조건 | 테스트 절차 | 예상 결과 |
| :--- | :--- | :--- | :--- | :--- |
| TC-01 | 앱 실행 | Node.js 설치 | 터미널에서 `npm start` 입력 | 브라우저가 열리고 메인 화면이 표시됨 |
| TC-02 | 메모 작성 | 앱 실행 중 | '메모를 입력하세요' 클릭 -> 내용 입력 -> 닫기 | 목록에 새 메모 카드가 생성됨 |
| TC-03 | 이미지 첨부 | 메모 작성 중 | 이미지 아이콘 클릭 -> 파일 선택 | 작성 창에 이미지 미리보기가 표시됨 |
| TC-04 | 메모 삭제 | 메모 존재 | 메모의 점 3개 메뉴 -> 삭제 클릭 | 메모가 메인 목록에서 사라지고 휴지통으로 이동 |
| TC-05 | 뷰 전환 | 메모 다수 존재 | 상단 뷰 토글 버튼 클릭 | 리스트 뷰와 그리드 뷰가 교차로 전환됨 |
| TC-06 | 검색 | 메모 존재 | 검색창에 특정 단어 입력 | 해당 단어가 포함된 메모만 필터링되어 표시됨 |

## 5. 버그 리포팅
-   발견된 버그는 우선순위(Critical, Major, Minor)를 지정하여 기록 및 수정.
